S4.4.1 Overview

Definition

  • Used to isolate client classes from derived classes

How does it work

  • Dynamic Binding we bind the f(x) to the appropriate f(x) during runtime0

Unbreakable rules of Polymorphism

  • only works on objects that are related to each other through inheritance
  • only works using pointers

Underlying idea

Example (pointers to objects)

class Animal {
    public:
        Animal(string n="") : name(n) {}
        void sing() { ... }
}

class Bird : public Animal {
    public:
        Bird(string n="") : Animal(n) {}
}

class Chicken : public Bird {
    public:
        Chicken(string n="") : Bird(n) {}
}

int main() {
    Animal  gertrude("Gertrude");
    Bird    birtrude("Birtrude");
    Chicken redHen("LittleRedHen");

    Animal*     ap;
    Bird*       bp;
    Chicken*    cp;

    cout << endl << "base ptr points to base object" << endl;
    // Base ptr points to base object
    // Derived ptr points to derived object (Bird, Chicken)
    ap = &gertrude; // Put animal pointer set to animal
    bp = &birtrude;
    cp = &redHen;

    ap -> sing();
    bp -> sing();
    cp -> sing();

    // Base ptr points to derived object
    // The hen is a type of bird so we can access the bird parts of the hen
    bp = &redHen;

    // Derived class ptr pointing to a derived object
    // Not allowed because it tries to do a conversion (down-cast) which is illegal
    // An animal is not a type of chicken
    cp = &gertrude;
    cp -> sing();
}

S4.4.3 Dynamic Binding

Terminology

  • handle is an identifier that is used to access an object
    • variable name
    • reference
    • pointer

Example Dynamic Binding (why it breaks)

class Animal {
    public:
        Animal(string n="") : name(n) {}
        void sing() {}
}

class Bird : public Animal {
    public:
        Bird(string n="") : Animal(n) {}
        void sing( cout << "tweet tweet" << endl; )
}

class Chicken : public Bird {
    public:
        Chicken(string n="") : Bird(n) {}
        void sing( cout << "cluck" << endl; )
}

class Cat : public Animal {
    public:
        Cat(string n="") : Animal(n) {}
        void sing( cout << "meow" << endl; )
}

// Static (regular) binding
int main() {
    Animal  gertrude("Gertrude");
    Bird    birtrude("Birtrude");
    Chicken redHen("LittleRedHen");
    Cat     lady("Lady");

    // Looks at the type of object and calls the sing function on that object
    gertrude.sing();
    birtrude.sing();
    redHen.sing();

    Animal*     ap;
    Animal*     ap2;
    Bird*       bp;
    Chicken*    cp;

    ap  = &birtrude;
    ap2 = &lady;
    bp = &redHen;

    // Unexpected behaviour
    // The pointers sing f(x) will call the base class f(x) since we access them with the animal pointer (so the animal f(x) version will be called)
    ap -> sing();
    ap2 -> sing();
    bp -> sing();
    // To fix this we need to use dynamic binding

    return 0;
}

Function Binding

Static Binding

  • selected at compile time
  • always used when called using object variable name

Dynamic Binding

  • selected at runtime
  • not used when called using object variable name
  • used when called using object pointer

Virtual Functions

  • Allows us to do dynamic binding
  • Makes selection based on the type of object and not the type handle
    • with a base class object the base class f(x) is called
    • with a derived class object the dereived class f(x) is called
  • Non-Virtual f(x)
    • f(x) selected at compile time

Example (Virtual Functions)

class Animal {
    public:
        Animal(string n="") : name(n) {}
        virtual void sing() {}
}

class Bird : public Animal {
    public:
        Bird(string n="") : Animal(n) {}
        virtual void sing( cout << "tweet tweet" << endl; ) // We use the virtual keyword so that we take advantage of polymorphism to bind the appropriate function
}

class Chicken : public Bird {
    public:
        Chicken(string n="") : Bird(n) {}
        virtual void sing( cout << "cluck" << endl; )
}

class Cat : public Animal {
    public:
        Cat(string n="") : Animal(n) {}
        virtual void sing( cout << "meow" << endl; )
}

Example (Bindings)

int main() {
    Animal  gertrudeObj("Gertrude");
    Bird    birtrudeObj("Birtrude");
    Chicken redHenObj("Little Red hen");
    Cat     ladyObj("Lady");

    vector<Animal> barnyardObj;

    barnyardObj.push_back(gertrudeObj);
    barnyardObj.push_back(birtrudeObj);
    barnyardObj.push_back(redHenObj);
    barnyardObj.push_back(ladyObj);

    cout << endl << "Barnyard harmony using objects..." << endl;
    
    for (int i = 0; i < barnyardObj.size(); ++i) {
        // Since we have a static binding here (called on the objects themselves) call the animal version of the sing() f(x)
        // Always static binding since we are calling it on the object
        barnyardObj[i].sing();
    }

    Animal*     gertrudePointer = new Animal("Gertrude");
    Bird*       birtrudePointer = new Bird("Birtrude");
    Chicken*    redHenPointer   = new Chicken("Little REd Hen");
    Cat*        ladyPointer     = new Cat("Lady");

    vector<Animal*> banyardPointers;

    banyardPointers.push_back(gertrudePointer);
    banyardPointers.push_back(birtrudePointer);
    banyardPointers.push_back(redHenPointer);
    banyardPointers.push_back(ladyPointer);

    for (int i = 0; i < barnyardPointers.size(); ++i) {
        // Prints out the expected result of birds chirp, etc...
        banyardObj[i] -> sing();
    }
}

Virtual Destructors

  • used to deallocate memory using the base class pointer (default calls the base class pointer)

Example Non-Virtual Destructors

class Animal {
    public:
        Animal(string n="") : name(n) {}
        ~Animal() { cout << "Animal is destroyed" << endl;}
        virtual void sing() {}
}

class Bird : public Animal {
    public:
        Bird(string n="") : Animal(n) {}
        ~Bird() { cout << "Bird is destroyed" << endl;} // We create a non-virtual desctructor
        virtual void sing( cout << "tweet tweet" << endl; )
}

class Chicken : public Bird {
    public:
        Chicken(string n="") : Bird(n) {}
        ~Chicken() { cout << "Chicken is destroyed" << endl;}
        virtual void sing( cout << "cluck" << endl; )
}

class Cat : public Animal {
    public:
        Cat(string n="") : Animal(n) {}
        ~Cat() { cout << "Cat is destroyed" << endl;}
        virtual void sing( cout << "meow" << endl; )
}

int main() {
    Animal  gertrudeObj("Gertrude");
    Bird    birtrudeObj("Birtrude");
    Chicken redHenObj("Little Red hen");
    Cat     ladyObj("Lady");

    vector<Animal> barnyardObj;

    barnyardObj.push_back(gertrudeObj);
    barnyardObj.push_back(birtrudeObj);
    barnyardObj.push_back(redHenObj);
    barnyardObj.push_back(ladyObj);

    cout << endl << "Barnyard harmony using objects..." << endl;
    
    for (int i = 0; i < barnyardObj.size(); ++i) {
        // This only calls the Animal destructor and not the individual destructors of the Bird, etc...
        delete barnyardObj[i];
    }
}

Example Virtual Destructors

class Animal {
    public:
        Animal(string n="") : name(n) {}
        virtual ~Animal() { cout << "Animal is destroyed" << endl;}
        virtual void sing() {}
}

class Bird : public Animal {
    public:
        Bird(string n="") : Animal(n) {}
        virtual ~Bird() { cout << "Bird is destroyed" << endl;} // We create a non-virtual desctructor
        virtual void sing( cout << "tweet tweet" << endl; )
}

class Chicken : public Bird {
    public:
        Chicken(string n="") : Bird(n) {}
        virtual ~Chicken() { cout << "Chicken is destroyed" << endl;}
        virtual void sing( cout << "cluck" << endl; )
}

class Cat : public Animal {
    public:
        Cat(string n="") : Animal(n) {}
        virtual ~Cat() { cout << "Cat is destroyed" << endl;}
        virtual void sing( cout << "meow" << endl; )
}

int main() {
    Animal  gertrudeObj("Gertrude");
    Bird    birtrudeObj("Birtrude");
    Chicken redHenObj("Little Red hen");
    Cat     ladyObj("Lady");

    vector<Animal> barnyardObj;

    barnyardObj.push_back(gertrudeObj);
    barnyardObj.push_back(birtrudeObj);
    barnyardObj.push_back(redHenObj);
    barnyardObj.push_back(ladyObj);

    cout << endl << "Barnyard harmony using objects..." << endl;
    
    for (int i = 0; i < barnyardObj.size(); ++i) {
        // Calls all of the objects destructors (bird will call the bird and animal destructors)

}

S4.4.4 Abstract Classes

Definition

  • a class that is general and can't have instances directly
  • no objects can be created of an abstract class
  • typically used a a base class
  • in C++ we do this by definining a pure virtual f(x)

Pure Virtual Function

  • a virtual function that does not have an implementation; = 0 in class definition
  • guarantees that all concrete classes must override it (implement it)
  • Having at least one pure virtual function makes a class abstract
Example
class Animal {
    public:
        Animal(string n="") : name(n) {}
        virtual ~Animal() { cout << "Animal is destroyed" << endl;}
        virtual void sing() {}
        virtual void dance() = 0; // We define this as a pure virtual f(x)
}

class Cat : public Animal {
    public:
        Cat(string n="") : Animal(n) {}
        virtual ~Cat() { cout << "Cat is destroyed" << endl;}
        virtual void sing() { ... };
        // Any derived class must implement the pure virtual f(x) dance
        virtual void dance() { ... };
}


// If we create a class that inherits from an abstract class (Animal) but doesn't implement the pure virtual f(x) then the created class also becomes an abstract class
class Pig : public Animal {
    public:
        Pig(string n="") : Animal(n) {}
        virtual ~Pig() { cout << "Cat is destroyed" << endl;}
        virtual void sing() { ... };
}
